# Task 4 – Static Dashboard Mock‑up & Insights Presentation

## Project Overview
Task 4 focuses on converting previous exploratory data analysis and SQL findings into a clear, visual **dashboard mock‑up** and an accompanying **insights presentation**. The goal is to practice business intelligence storytelling by highlighting the most important KPIs, charts, and interpretations rather than doing new heavy analysis. [file:1]

## Objective
- Create a static dashboard layout (in PowerPoint, Google Slides, Excel, or any visualization tool) using the insights from earlier tasks.
- Present key business metrics and trends in a single consolidated view.
- Explain the main insights and recommendations in a short presentation or video (5–7 minutes). [file:1]

## Data & Source Tasks
This task builds directly on the dataset and work from:
- **Task 2 / 3 – Exploratory Data Analysis & SQL**  
  - Descriptive statistics and univariate analysis  
  - SQL queries for business questions  
  - Multivariate analysis and correlation  
These earlier steps provide the metrics and charts that are summarized in this dashboard. [file:1][web:150]

## KPIs Included
The dashboard highlights a small set of core KPIs:
- **Total Revenue** – overall business performance.
- **Total Orders** – count of transactions.
- **Total Customers** – distinct customer count.
- **Average Order Value (AOV)** – revenue per order.
- **Top Category / Top City** – highest‑performing segments. [web:148]

These KPIs are placed as cards at the top of the dashboard to give a quick summary of performance.

## Dashboard Layout
The static dashboard mock‑up contains the following visual elements: [web:148][file:1]

1. **KPI Card Row**
   - Total Revenue  
   - Total Orders  
   - AOV  
   - Total Customers  

2. **Charts**
   - **Monthly Sales Trend** (line or bar chart) – shows revenue over time.
   - **Revenue by Product Category** (bar chart).
   - **Sales by City** (bar chart or map‑style chart).
   - **Payment Method Distribution** (donut/pie chart).
   - **Top Products by Revenue** (horizontal bar chart).

3. **Filters / Slicers (Optional for mock‑up)**
   - Date range
   - Product category
   - City
   - Payment method

The layout is designed so that a manager can understand the business status in one glance while still being able to drill into individual charts. [web:148]

## Key Insights (Example)
These are sample insights to document; replace with your actual numbers:

- A small number of product categories contribute most of the revenue.
- One or two cities account for a large share of sales.
- Monthly revenue shows a noticeable upward/downward trend.
- Customers strongly prefer certain payment methods.
- Average order value remains stable (or increasing/decreasing) across the period.

Each insight should be tied to a specific chart or KPI on the dashboard.
